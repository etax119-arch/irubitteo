# 직원 대시보드 기능 명세

> 참조: `durubitteo_design/src/AttendanceApp.jsx`

## 개요

직원(장애인 근로자)이 사용하는 출퇴근 앱으로, 간단한 UI와 큰 글씨로 접근성을 고려한 설계.

---

## 구현 상태

**구현 완료**

### 라우트 구조

```
/app/employee/
├── layout.tsx              # 직원용 레이아웃
├── page.tsx                # 메인 화면 (출근/퇴근 버튼)
├── checkin/page.tsx        # 출근 화면
├── checkout/page.tsx       # 퇴근 화면
├── _components/
│   ├── HeaderCard.tsx          # 사용자 인사 + 날짜 + 로그아웃
│   ├── AttendanceButtons.tsx   # 출근/퇴근 버튼 그리드
│   ├── NoticeSection.tsx       # 긴급 공지 섹션
│   ├── WorkRecordsSection.tsx  # 활동 기록 아코디언
│   ├── DateNavigator.tsx       # 연도/월 네비게이션
│   ├── WorkRecordCard.tsx      # 활동 기록 카드
│   ├── PhotoLightbox.tsx       # 사진 확대 모달
│   ├── HeicImage.tsx           # HEIC 이미지 지원
│   └── SuccessModal.tsx        # 출퇴근 완료 모달
└── _hooks/
    └── useWorkRecords.ts       # 활동 기록 상태 관리
```

### URL 구조

| URL | 설명 |
|-----|------|
| `/login/employee` | 직원 로그인 (고유번호 입력) |
| `/employee` | 메인 화면 (출근/퇴근 버튼) |
| `/employee/checkin` | 출근 화면 |
| `/employee/checkout` | 퇴근 화면 |

---

## 화면 구성

| 화면 | 설명 |
|------|------|
| 로그인 | 고유번호 입력 (4-6자리 영숫자) |
| 메인 | 출근/퇴근 버튼, 긴급 공지 섹션 |
| 출근 | 오늘의 작업 내용 확인 → 출근 완료 |
| 퇴근 | 업무 내용 입력, 사진 첨부 → 퇴근 완료 |

---

## 화면별 상세

### 1. 로그인 화면

**목적**: 근로자 본인 확인

**UI 요소**:
- 고유 번호 입력 필드
  - 4-6자리 영숫자 허용
  - 예시: "JH1234"
  - 큰 글씨, 중앙 정렬, 추적 간격 적용
- "출퇴근 시작" 버튼 (유효한 번호 입력 시 활성화)
- "고유 번호를 모르시나요? 관리자에게 문의해주세요" 안내 문구
- "메인으로 돌아가기" 버튼

**동작**:
1. 고유 번호 입력
2. 4-6자리 입력 시 버튼 활성화
3. 버튼 클릭 또는 Enter 키 → 메인 화면 이동
4. 서버에서 사용자 정보(이름) 조회

---

### 2. 메인 화면

**목적**: 출근/퇴근 선택 및 긴급 공지 확인

**UI 요소**:
- 상단 인사 영역
  - "{이름}님, 환영합니다!"
  - 현재 날짜 표시 (예: "2026년 1월 28일 수요일")
  - 홈 버튼 (메인 랜딩으로 이동)
- 출근/퇴근 버튼 (2열 그리드)
  - **출근하기**: 주황색 그라데이션 버튼
    - 아이콘: Clock
    - 설명: "오늘의 할 일을 확인하세요"
  - **퇴근하기**: 회색 그라데이션 버튼
    - 아이콘: CheckCircle2
    - 설명: "오늘 한 일을 기록하세요"

**긴급 공지 섹션**:
- **금일 긴급 공지** (강조 영역)
  - 주황색 배경, 큰 알림 아이콘
  - 공지 날짜, 내용, 전송자 표시
  - 공지가 없을 경우 "금일 등록된 긴급 공지가 없습니다" 표시
- **지난 긴급 공지** (아코디언)
  - "모두 보기" / "접기" 토글
  - 펼치면 과거 공지 목록 표시
  - 각 공지: 날짜, 내용, 전송자

---

### 3. 출근 화면

**목적**: 오늘의 작업 내용 확인 후 출근 처리

**UI 요소**:
- 뒤로가기 버튼
- 타이틀: "출근하기"
- 설명: "오늘 할 일을 확인해주세요"

**오늘의 작업 내용 섹션**:
- 연한 주황색 배경
- 기업 관리자가 근무일정관리에서 등록한 작업 내용 표시
- 예시:
  ```
  오늘은 제품 포장 작업과
  부품 조립 작업을 진행할 예정입니다.
  작업 후에는 작업장 정리정돈을 함께 해주세요.
  안전하게 천천히 진행하시면 됩니다.
  ```

**확인 체크 영역**:
- 체크박스: "오늘 할 일을 확인했어요!"
- 체크해야 출근 완료 버튼 활성화

**출근 완료 버튼**:
- 체크박스 선택 시 활성화
- 클릭 시 출근 완료 모달 표시

**출근 완료 모달**:
- 스마일 아이콘
- "출근이 완료되었어요! 오늘도 파이팅!"
- "안전하게 근무하시길 바랍니다."
- "메인으로 이동" 버튼

---

### 4. 퇴근 화면

**목적**: 오늘의 업무 내용 기록 후 퇴근 처리

**UI 요소**:
- 뒤로가기 버튼
- 타이틀: "퇴근하기"
- 설명: "오늘 한 일을 간단히 기록해주세요"

**오늘의 업무 내용 입력**:
- 텍스트 영역 (5줄)
- 플레이스홀더: "예) 제품 포장 작업, 부품 조립, 작업장 정리 등"
- **필수 입력** - 입력해야 퇴근 완료 버튼 활성화

**사진 첨부 섹션**:
- 연한 주황색 배경
- 안내: "자택 근무 시 활동 사진을 꼭 함께 업로드해 주세요!"
- 부가 안내: "*사진 첨부는 필수가 아닙니다."
- "사진 추가하기" 버튼 (파일 선택)
- 첨부된 사진 미리보기 (썸네일 + 삭제 버튼)

**퇴근 완료 버튼**:
- 업무 내용 입력 시 활성화
- 클릭 시 퇴근 완료 모달 표시

**퇴근 완료 모달**:
- 스마일 아이콘
- "퇴근 완료!"
- "오늘도 너무 고생하셨습니다"
- "편안한 저녁 시간 보내세요."
- "메인으로 이동" 버튼

## 데이터 흐름

### 입력 데이터
| 항목 | 출처 | 설명 |
|------|------|------|
| 고유 번호 | 사용자 입력 | 로그인용 4-6자리 코드 |
| 오늘의 작업 내용 | 서버 (기업 관리자 등록) | 출근 시 확인할 작업 안내 |
| 긴급 공지 | 서버 (기업 관리자 발송) | 금일/과거 긴급 공지 |

### 출력 데이터
| 항목 | 대상 | 설명 |
|------|------|------|
| 출근 기록 | 서버 | 출근 시간, 작업 확인 여부 |
| 퇴근 기록 | 서버 | 퇴근 시간, 업무 내용, 첨부 사진 |

---

## 접근성 고려사항

- 큰 글씨 (제목 2xl~3xl, 본문 lg~xl)
- 큰 버튼 (py-4~5, 최소 터치 영역 확보)
- 명확한 색상 대비 (주황/흰색)
- 간단한 흐름 (최소한의 단계)
- 체크박스와 레이블 연결

---

## 연관 기능

- 기업 대시보드 > 근무일정관리: 오늘의 작업 내용 등록
- 기업 대시보드 > 공지사항: 긴급 공지 발송
- 기업 대시보드 > 대시보드: 출퇴근 기록 조회
